# <a name="chapter-1---introduction-to-azure-rtos-guix"></a>Глава 1. Знакомство с GUIX для ОСРВ Azure

GUIX для ОСРВ Azure — это высокопроизводительная, работающая в реальном времени реализация платформы графического интерфейса, разработанная исключительно для встраиваемых приложений на основе ThreadX. В этой главе содержатся общие сведения о GUIX, в том числе описываются аспекты его прикладного применения и преимущества.

## <a name="guix-feature-overview"></a>Обзор возможностей GUIX

В отличие от многих других реализаций графического пользовательского интерфейса, GUIX обладает гибкостью — его можно легко использовать как с компактными приложениями для микроконтроллеров, так и с приложениями для мощных процессоров RISC и DSP. Это резко контрастирует с общедоступными или другими коммерческими реализациями, которые изначально разрабатывались для сред рабочих станций, а затем адаптировались под встраиваемые системы. Ниже приводится обзор возможностей GUIX:

- Простота в работе благодаря размещенному инструменту проектирования GUIX Studio

- Среда выполнения Win32 GUIX для полностью размещенного создания прототипов

- Поддержка большинства процессоров, поддерживаемых ThreadX

- Написание исключительно на ANSI C

- Независимость от порядка байтов

- Самый маленький и быстрый встраиваемый графический пользовательский интерфейс

- Возможность настройки количества объектов, размера экрана и других параметров во время выполнения

- Простота написания интерфейса для драйвера дисплея

- Поддержка цветного (глубина цвета до 32 бит/пкс), монохромного режимов и режима оттенков серого

- Многоязычная поддержка с использованием кодирования строк UTF8 и строковых ресурсов

- Бесплатные шрифты по умолчанию и простота добавления новых шрифтов

- Поддержка нескольких холстов разного размера

- Поддержка нескольких экранов разного размера с разными параметрами глубины цвета

- Поддержка экранных переходов (появление, исчезание, прокрутка и т. д.)

- Поддержка сенсорного ввода, жестов и виртуальной клавиатуры

- Сжатие растрового изображения

- Поддержка альфа-смешения

- Поддержка размывания

- Поддержка сглаживания

- Создание обложек и тем

- Смешение на холсте

- Полное управление окнами

  - Связи между родительскими и дочерними объектами

  - Динамическое создание, удаление, изменение размера и перемещение
  - Разделение обработки событий и отрисовки 
  - Z-порядок
  - Обрезка и представления

- Обширный набор мини-приложений

  - Различные типы кнопок, ползунков и шкал

  - Раскрывающийся список
  
  - prompt

  - Представление многострочного текста
  
  - Одиночный и многострочный ввод текста
  
  - Числовые и текстовые колесики прокрутки
  
  - Окна и полосы прокрутки
  
  - Круговой индикатор выполнения
  
  - Спрайт

### <a name="ansi-c-source-code"></a>Исходный код ANSI C

GUIX написан полностью на ANSI C, и его можно мгновенно перенести практически в любую процессорную архитектуру с поддержкой компилятора ANSI C и ThreadX. Несмотря на то, что GUIX написан на ANSI C, в нем используются объектно-ориентированная модель и наследование.

### <a name="not-a-black-box"></a>Отсутствие проблемы "черного ящика"

Большинство дистрибутивов GUIX включают полный исходный код на C. Это устраняет проблему "черного ящика", которая характерна для многих коммерческих реализаций графического пользовательского интерфейса. Используя GUIX, разработчики приложений могут отслеживать все без исключения операции графического пользовательского интерфейса.

Доступность исходного кода также позволяет вносить изменения с учетом конкретного приложения. Хотя это и не рекомендуется, но возможность при необходимости изменить графический пользовательский интерфейс является плюсом. Эти функции особенно полезны разработчикам, которые привыкли работать с корпоративными или общедоступными продуктами. Они привыкли, что исходный код доступен и его можно изменить. GUIX — это лучшее программное обеспечение графического пользовательского интерфейса для таких разработчиков.

## <a name="embedded-gui-applications"></a>Встраиваемые приложения с графическим пользовательским интерфейсом

Встраиваемые приложения с графическим пользовательским интерфейсом — это приложения, которым требуется пользовательский интерфейс и которые работают на микропроцессорах, установленных в таких продуктах, как сотовые телефоны, коммуникационное оборудование, двигатели, лазерные принтеры, медицинские изделия и т. д. Такие приложения почти всегда имеют некоторые ограничения по объему памяти и производительности. Еще одно отличие встраиваемых приложений с графическим пользовательским интерфейсом — это то, что их программные и аппаратные компоненты имеют определенные цели.

### <a name="real-time-gui-software"></a>Программное обеспечение реального времени с графическим пользовательским интерфейсом

Программное обеспечение с графическим пользовательским интерфейсом, которое должно выполнять операции обработки за определенный период времени, называется *ПО реального времени с графическим пользовательским интерфейсом*, и при наличии временных ограничений для таких приложений они считаются приложениями реального времени. Встраиваемые приложения с графическим пользовательским интерфейсом практически всегда работают в реальном времени, так как они взаимодействуют с внешней средой.

## <a name="guix-benefits"></a>Преимущества GUIX

Основными преимуществами использования GUIX для встраиваемых приложений являются высокая производительность, широкий выбор возможностей и небольшие требования к памяти. Кроме того, протокол GUIX полностью интегрирован с высокопроизводительной, многозадачной операционной системой ОСРВ Azure ThreadX, работающей в реальном времени.

### <a name="improved-responsiveness"></a>Повышенная скорость реагирования

Благодаря высокой производительности GUIX приложения реагируют быстрее, чем когда-либо ранее. Это особенно важно для встраиваемых приложений, которые работают со значительным объемом визуальных данных или имеют строгие ограничения по времени, затрачиваемому на отображение такой информации.

### <a name="software-maintenance"></a>Обслуживание программного обеспечения

С GUIX разработчики могут с легкостью секционировать аспекты собственных встраиваемых приложений, связанные с графическим пользовательским интерфейсом. Такое секционирование упрощает весь процесс разработки и значительно расширяет возможности по будущему обслуживанию программного обеспечения.

### <a name="increased-throughput"></a>Повышение пропускной способности

GUIX предоставляет графический интерфейс пользователя с наивысшей возможной производительностью напрямую встраиваемому приложению. Приложения GUIX могут обрабатывать данные пользовательского интерфейса быстрее любых других приложений.

### <a name="processor-isolation"></a>Изолированность от процессора

GUIX предоставляет надежный, независимый от процессора интерфейс между приложением, базовым процессором и дисплеем. Это позволяет разработчикам сосредоточиться на высокоуровневых аспектах пользовательского интерфейса, не тратя время на решение вопросов, связанных с оборудованием.

### <a name="ease-of-use"></a>Удобство использования

GUIX предназначается для разработчиков приложений. Архитектура и интерфейс вызовов служб GUIX отличаются простотой. Поэтому разработчики GUIX могут с легкостью использовать его расширенные функции.

### <a name="improve-time-to-market"></a>Сокращение времени выхода на рынок

Мощные функции GUIX ускоряют разработку программного обеспечения. GUIX абстрагирует большинство проблем с процессором и дисплеем, фактически устраняя их в большинстве реализаций пользовательского интерфейса приложения. В сочетании с простотой использования и набором расширенных функций это значительно ускоряет выход на рынок.

### <a name="protecting-the-software-investment"></a>Защита инвестиций в программное обеспечение

Для разработки GUIX используется исключительно ANSI C. При этом он полностью интегрирован с операционной системой ОСРВ Azure ThreadX, работающей в реальном времени. Это означает, что приложения GUIX можно мгновенно перенести на все поддерживаемые ThreadX процессоры. Более того, поддержку новой процессорной архитектуры в ThreadX можно реализовать всего за несколько недель. В результате использование GUIX обеспечивает беспроблемную миграцию приложения и защищает изначальные инвестиции в разработку.
