## <a name="event-descriptions"></a>Описания событий

### <a name="internal-thread-resume"></a>Возобновление внутреннего потока

#### <a name="internal-thread-resume"></a>Возобновление внутреннего потока

**Значок** ![Значок возобновления внутреннего потока](./media/user-guide/tx-events/image1.png)

**Описание**

Это событие представляет внутреннюю обработку в ThreadX, которая возобновляет поток для выполнения. Если указанный поток обладает наивысшим приоритетом, а его выполнение не блокируется из-за превышения порогового значения для вытеснения, система начнет выполнение этого нового готового потока.

**Поля сведений**

- Поле сведений 1. Указатель на возобновляемый поток.
- Поле сведений 2. Предыдущее состояние возобновляемого потока (см. таблицу ниже).

  |  Состояние потока                     | Значение        |
  |---------------------------------- | --------|
  |  TX_READY                         | 0       |
  |  TX_COMPLETED                     | 1       |
  |  TX_TERMINATED                    | 2       |
  |  TX_SUSPENDED                     | 3       |
  |  TX_SLEEP                         | 4       |
  |  TX_QUEUE_SUSP                    | 5       |
  |  TX_SEMAPHORE_SUSP                | 6       |
  |  TX_EVENT_FLAG                    | 7       |
  |  TX_BLOCK_MEMORY                  | 8       |
  |  TX_BYTE_MEMORY                   | 9       |
  |  TX_TCP_IP                        | 12      |
  |  TX_MUTEX_SUSP                    | 13      |

- Поле сведений 3. Значение указателя стека во время вызова. 
- Поле сведений 4. Указатель на следующий поток с наивысшим приоритетом для выполнения.

### <a name="internal-thread-suspend"></a>Приостановка внутреннего потока

#### <a name="internal-thread-suspend"></a>Приостановка внутреннего потока

**Значок** ![Значок приостановки внутреннего потока](./media/user-guide/tx-events/image2.png)

**Описание**

Это событие представляет внутреннюю обработку в ThreadX, которая приостанавливает выполнение потока. В четвертое поле сведений помещается следующий готовый к выполнению поток с наивысшим приоритетом. Если это значение равно NULL, другие готовые к выполнению потоки отсутствуют, поэтому система бездействует.

**Поля сведений**

- Поле сведений 1. Указатель на приостановленный поток.
- Поле сведений 2. Новое состояние приостановленного потока (см. таблицу ниже).

  |  Состояние потока                     | Значение        |
  |---------------------------------- | --------|
  |  TX_COMPLETED                     | 1       |
  |  TX_TERMINATED                    | 2       |
  |  TX_SUSPENDED                     | 3       |
  |  TX_SLEEP                         | 4       |
  |  TX_QUEUE_SUSP                    | 5       |
  |  TX_SEMAPHORE_SUSP                | 6       |
  |  TX_EVENT_FLAG                    | 7       |
  |  TX_BLOCK_MEMORY                  | 8       |
  |  TX_BYTE_MEMORY                   | 9       |
  |  TX_TCP_IP                        | 12      |
  |  TX_MUTEX_SUSP                    | 13      |

- Поле сведений 3. Значение указателя стека во время вызова. Поле сведений 4. Указатель на следующий поток с наивысшим приоритетом для выполнения. Если значение равно NULL, система бездействует.

### <a name="interrupt-service-routine-isr-enter"></a>Вход в подпрограмму обработки прерываний (ISR) 

#### <a name="enter-isr"></a>Вход в ISR 

**Значок** ![Значок входа в ISR](./media/user-guide/tx-events/image3.png)

**Описание**

Это событие представляет вход в подпрограмму обработки прерываний (ISR) в приложении. Выполнение подпрограммы обработки прерываний продолжается до наступления события выхода из ISR.

**Поля сведений**

- Поле сведений 1. Значение указателя стека во время вызова.
- Поле сведений 2. Определяемый приложением номер ISR (необязательно).
